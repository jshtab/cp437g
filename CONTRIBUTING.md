 Non-negotiable requirements
=============================

All incoming branches must, at a minimum, meet these requirements:

 - Tests must pass in all supported Python versions.
 - No dependencies except the Python standard library.
 - Introductory messages (the PR description) must:
    - Be written in English,
    - Explain what was changed, and
    - Justify why the branch should be pulled.

 Would-be-nice-to-haves
========================

It would make me very happy if you:

 - Wrote useful commit messages that don't need to be amended in a rebase.
 - Wrote Markdown files (like the README) that are readable as plain text.
 - Didn't generate your introductory message or code using a large language model
 - Didn't make pull requests automatically through your agents
